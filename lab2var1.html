For each instruction, one prefix may be used from each of these group and placed in any order. Using redundant prefixes (more than one prefix from a group) is reserved and may cause unpredictable behavior.

The LOCK prefix forces an atomic operation to insure exclusive use of shared memory in a multiprocessor environment. See "LOCK-Assert LOCK# Signal Prefix" in Chapter 3, Instruction Set Reference, for a detailed description of this prefix and the instruction with which it can be used.

The repeat prefixes cause an instruction to be repeated for each element of a string. They can be used only with the string instruction: MOVS, CMPS, SCAS, LODS, STOS, INS, and OUTS. Use behavior (see the note below)

They branch hint prefixes allow a program to give a hint to the processor about the most likely code path that will be taken at a branch. These prefixes can only be used with the conditional branch instruction (Jcc). Use of these prefixes with other IA-32 instruction is reserved and may cause unpredictable behavior. The branch hint prefixes were introduced in the Pentium 4 and Intel Xeon processors as part the SSE2 extensions.
